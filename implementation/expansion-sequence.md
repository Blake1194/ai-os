# Expansion Sequence

## What Comes First
1. open controlled expansion mode in the control/state docs
2. create the second-generation product shell scaffold that also prepares the Android/mobile lane
3. prepare the install-capable workspace around that scaffold
4. prove live runtime behavior in the simplest real way possible

## What Comes Second
5. add the first interactive local/device runtime path
6. validate direct computer access behind explicit capability gates
7. only then begin the first integrations that materially help the shell

## What Comes Third
8. add voice interaction
9. add bounded multi-agent worker orchestration
10. add selected high-value integrations such as Shopify and Slack
11. add optional front-end/chatbot surfaces

## What Depends On What
- the next-generation shell depends on a clear architecture direction first
- actual mobile build/run work depends on a real install-capable toolchain
- direct full-computer access depends on a stable local runtime and capability gateway
- meaningful external integrations depend on the shell/runtime being stable enough to receive them
- multiple workers depend on clear supervisory control and stop-gates
- voice and optional camera/context access depend on explicit consent/override rules
- Shopify acceleration should follow a stable assistant baseline rather than replace it

## What Should Not Start Yet
- live credentialed integrations before the shell baseline exists
- destructive migrations away from the working V1 baseline
- multiple parallel architecture experiments with no integration point
- Android packaging and device deployment before toolchain bootstrap
- uncontrolled self-copying or silent full-device autonomy

## What Would Create Chaos If Done Too Early
- starting several integration lanes before the product shell/runtime exists
- adding computer-wide access before the consent/override model is real
- adding multi-agent workers before the supervisor/stop-gate model is real
- attempting voice/camera layers without visible permissions and hard revoke controls
- mixing destructive refactors into expansion without clear checkpoints
