# Task Execution Workflow

## Purpose
Define how one real task moves through Atlas from start to finish.

## Sequence
1. Receive task
2. Classify task type
3. Determine whether it is planning, execution, research, or mixed
4. Identify blockers and dependencies
5. Separate reversible vs irreversible actions
6. Execute only safe reversible steps
7. Stop before irreversible steps if approval is needed
8. Record actions taken
9. Record decisions made
10. Recommend the next action

## Workflow Stages
### Task Intake
- capture the request
- identify the objective
- identify source and destination if relevant

### Scope Clarification
- clarify only if a real blocker exists
- keep the task bounded and practical

### Planning
- choose the best tool or page
- define the shortest safe path

### Safe Execution
- inspect the page
- navigate
- open forms or drafts
- fill obvious fields
- prepare work to the review point

### Review
- check what is ready
- identify the exact irreversible step not taken

### Logging
- record what was done
- record what changed
- record what still needs approval

### Follow-Up
- recommend the next best action
- capture reusable patterns as commands or workflows

## Example Reversible Actions
- opening tabs or pages
- navigating menus
- drafting content
- filling non-final fields
- selecting obvious options
- copying visible information

## Example Irreversible Actions
- submitting a form
- publishing content
- sending email or messages
- deleting data
- confirming purchases
- saving live changes with no easy rollback

## Required Response / Report Format
- goal
- page/context
- action taken
- prepared status
- stopped before
- waiting on (only if blocked)

## Successful Task Run
A task run is successful when Atlas:
- understands the task
- performs all safe steps it can
- stops at the right approval point
- records useful context for the next step
