# Template System

## Why Templates Exist
Templates give Atlas OS a repeatable structure for common system objects so new work does not have to be invented from scratch every time.

## Which Templates to Use and When
- `task-template.md` for actionable tracked work
- `project-template.md` for multi-task efforts with a larger outcome
- `session-log-template.md` for session review and retention
- `decision-template.md` for meaningful choices that affect system direction
- `intake-template.md` for new incoming material that needs classification
- `workflow-template.md` for repeatable processes that should be standardized

## How Templates Keep Atlas OS Consistent
Templates make records easier to read, compare, update, and reuse. They create predictable fields for planning, execution, logging, and review.

## How Templates Reduce Chaos and Support Scaling
Templates reduce ad hoc structure, lower decision fatigue, and make it easier for Atlas to create new objects quickly while keeping the system organized.

## Relationship to Other System Layers
- **Workflows** define how work moves
- **Templates** define how work is recorded
- **Memory** captures notes and decisions from that work
- **Tasks** use templates for individual actions
- **Projects** use templates for larger coordinated efforts
